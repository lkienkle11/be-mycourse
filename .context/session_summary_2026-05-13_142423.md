## Key decisions
- Use double-submit cookie (`csrf_token`) + header (`X-CSRF-Token`) for state-changing requests.
- Provide explicit FE bootstrap endpoint (`GET /api/v1/auth/csrf`) so FE can fetch token before POST/PATCH/PUT/DELETE calls.
- Enforce CSRF on unsafe methods in both unauthenticated and authenticated `/api/v1` lanes (GET/HEAD/OPTIONS/TRACE excluded).
