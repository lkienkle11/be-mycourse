- **Additive fix only**: Signature/behavior of `ImageSizeFromPayload` unchanged — `(0,0)` still returned for empty/corrupted payloads; only WebP now decodes correctly.
- **No fallback path**: Pipeline rejects image uploads if WebP encode fails (errors bubble up from `encodeUploadToWebP`); there is no silent fall-through to original payload.
