## Overview

Fixed Bunny Stream video upload returning `width=0, height=0` for both CREATE and UPDATE responses. Root cause: `ApplyBunnyDetailToMetadata` only wrote `video_id`, `thumbnail_url`, `embeded_html` — silently discarding the video telemetry fields (`width`, `height`, `length`, `framerate`, `bitrate`, `video_codec`, `audio_codec`) available in `BunnyVideoDetail`. Added helper `applyBunnyVideoTelemetry` to write all seven fields when non-zero, unifying upload path and webhook path behavior.

## Key decisions

- **Non-zero guard**: Only write telemetry fields when `> 0` / non-empty so partial metadata maps are safe to merge.
- **Residual 0 after immediate upload**: If Bunny hasn't finished transcoding at the time of the immediate post-upload `GetBunnyVideoByID` call, width/height will still be 0. The Bunny webhook (`HandleBunnyVideoWebhook`) updates them once transcoding completes — this is intentional/expected behavior documented in `docs/modules/media.md`.
- **Unified key names**: Both upload path and webhook path now use identical keys (`width`, `height`, `length`, `framerate`, `bitrate`, `video_codec`, `audio_codec`) so `buildVideoTypedMetadata` reads them correctly in both paths.
