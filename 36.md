# Tong hop codebase `openedu-core`

## 1) Tong quan nhanh

- `openedu-core` la backend API monolith viet bang Go, dung Gin + GORM + PostgreSQL, Redis, RabbitMQ.
- Entry point: `main.go`.
- Kien truc theo lop ro rang:
  - **Presentation**: `api/`, `middleware/`, `docs/`
  - **Business**: `services/`, `dto/`, `queues/`
  - **Data**: `models/`, `cache_clients/`, `models/migrations/`
  - **Infrastructure**: `pkg/`, `config/`, `tracing/`, `runtime/`
- So luong file chinh (scan theo cau truc hien tai):
  - `api/v1`: 74 files endpoint
  - `api/config`: 16 files ACL/permission bootstrap
  - `services`: 106 files
  - `models`: 143 files
  - `dto`: 81 files
  - `models/migrations`: 58 files SQL migration

## 2) Cau truc thu muc va vai tro

- `api/router.go`: map route cho `public`, `api/v1`, `api/internal-v1`.
- `api/v1/*.go`: handler theo domain (`auth`, `course`, `payment`, `blog`, `form`, `ai`, `launchpad`, `oe-referral`...).
- `middleware/middleware.go`: authn/authz, domain resolution, permission gate, context injection.
- `services/*.go`: business logic, orchestration, transaction boundaries, external integrations.
- `models/*.go`: repository + entities + query structs + migration bootstrap.
- `queues/queues.go`: worker consumers cho RabbitMQ.
- `pkg/*`: adapter/integration (AI, queue, payment, upload, communication, log, metrics, scheduler...).
- `config/*`: file cau hinh `.ini` theo moi truong + bootstrap init system.
- `tracing/`: Grafana/Prometheus provisioning.
- `.data_storage/`: local persisted data cho Mongo/Postgres/Redis/Rabbit khi chay local docker (du lieu runtime).

## 3) API layer

### 3.1 Group route

- **Public route** (khong qua auth middleware day du):
  - `/api/v1/health`
  - callback payment (`vnpay`, `momo`)
  - cert image, usage ai post, wallet tra cuu, leaderboard public...
- **Main app route**: `/api/v1/*`
  - qua `BeforeInterceptor()` de check header, token, permission, org context.
  - cac nhom lon: `auth`, `users`, `courses`, `lessons`, `quizzes`, `orders`, `payments`, `blogs`, `forms`, `approvals`, `reports`, `cron-jobs`, `ai`, `launchpads`, `oe-referrals`, `schedules`...
- **Internal route**: `/api/internal-v1/*`
  - cho service-to-service (role/user/org/plan/resource usage...).

### 3.2 ACL va permission

- Permission duoc migrate/init tu `api/config/*` va `config.InitSystem()`.
- Middleware lay `method + fullPath + handlerName` de lookup access matrix.
- Co role guest cho endpoint public; endpoint private can token + role mapping.

## 4) Data flow (request flow + background flow)

### 4.1 Request flow HTTP

1. Request vao Gin router (`api.InitRouter()`).
2. Qua middleware:
   - logging/recovery/gzip/cors
   - `BeforeInterceptor()`:
     - doc `Authorization`, `Origin`, `X-Referer`, `X-API-Key`
     - parse domain + map organization
     - parse JWT (neu co), check OTP flag, user active/blocked
     - check permission theo role access
3. Handler `api/v1/*.go` bind request DTO.
4. Goi service `services.*` xu ly nghiep vu.
5. Service goi repository `models.Repository.*` + cache + external pkg.
6. Tra response standardized qua package app/e.

### 4.2 Background/async flow

- `main.go` start queue consumer song song HTTP server.
- `queues.Consume()` subscribe RabbitMQ theo prefix env (`local_`, `staging_`, `prod_`...).
- Queue hien tai co worker pools cho:
  - `wallet_sync_queue`
  - `wallet_retrieve_get_details_queue`
  - `transaction_sync_queue`
  - `clp_launchpad_job_sync_queue`
- Worker handlers nam trong `queues/handlers/*`.

### 4.3 Startup flow

`main.go` init theo thu tu:
- `setting.Setup()` -> load INI
- setup cache/log/db/upload/communication/exchange/chain/services/validator/AI/scheduler
- `models.MigrateDatabase()` (chi chay khi `MIGRATE=1`)
- `config.InitSystem()` seed roles, permission, default account/org/forms/config/plans/ai model...

## 5) Data layer va persistence

- DB: PostgreSQL qua GORM (`models.Setup()`), co naming strategy + callbacks timestamp/id.
- Multi-tenant schema logic:
  - `models.AppSchema` set theo organization trong middleware.
  - Shared tables dung `public`, org-specific tables dung schema org.
- Redis:
  - 1 client global (`models.Redis`)
  - cache abstraction trong `cache_clients/*`.
- Repository pattern:
  - `models.Repository` aggregate rat nhieu repository con (user, course, payment, ai, referral...).

## 6) Config files (doc chi tiet)

## 6.1 Co che load config

- File loader nam o `pkg/setting/setting.go`.
- Thu tu chon file:
  - `APP_ENV_FILE` (neu set) uu tien cao nhat.
  - Neu `STAGE` rong -> `config/app.ini`
  - Neu `STAGE=dev|staging|test|prod|local|...` -> `config/app-{stage}.ini`
  - Neu `TEST=true` -> `../config/app.ini`
- Map section vao struct global: `AppSetting`, `ServerSetting`, `DatabaseSetting`, ...
- Tu dong parse cac section:
  - `google_form_default`
  - tat ca section co prefix `google_sheet_` vao `GoogleSheetSettings`.

## 6.2 Danh sach file config

- `config/app.ini` (local/default khi khong set `STAGE`)
- `config/app-local.ini`
- `config/app-dev.ini`
- `config/app-staging.ini`
- `config/app-test.ini`
- `config/app-prod.ini`
- `config/bootstrap.go` + `config/default_system_configs.go` (seed runtime configs)

## 6.3 Cac section config chinh (xuat hien o hau het env)

- `[server]`: run mode, host, port, timeout, host domain, build name.
- `[database]`: postgres connection + table prefix + ssl mode.
- `[app]`:
  - JWT secrets/expire, ApiKey, AIApiKey
  - domain/path mapping (`AdminHost`, `BaseDomain`, `OrgAdminPath`, `PartnerPath`, `LearnerPath`, `BlogPath`, `AffiliatePath`)
  - default sysadmin account
  - log/image/runtime controls
  - alert toggles (`EnableAPIFailureAlerts`, `EnableAIFailureAlerts`, webhook teams)
  - `CouponReserveDuration`
- `[redis]`: addr/password/pool/channel prefix.
- `[aws]`: S3 credentials/region.
- `[email]`: provider + sender + api key + redirect uri.
- `[upload]`: provider (`s3`), bucket/path, Bunny configs.
- `[google]`, `[facebook]`: OAuth config.
- `[payment]`: Sepay QR URI template.
- `[external-service]`: AI core domain + x-api-key + leaderboard endpoint.
- `[openedu-communication]`: internal communication service domain + api key.
- `[rabbitmq]`: broker URL + prefix.
- `[openedu-chain]`: blockchain service config + `IsMainnet`.
- `[coin-gecko]`, `[openexchangerates]`, `[coinmarketcap]`.
- `[openedu-scheduler]`: scheduler service domain + api key.
- `[vnpay]`, `[vnpay.course]`, `[vnpay.aimodel]`: payment gateway va merchant-specific credentials.
- `[google_form_default]` + cac `[google_sheet_*]`: ket noi Google Form/Sheet.
- `[momo]`: config payment gateway MoMo.
- `[ai-nextgen]`: secret + redirect base (co mat trong `app-dev.ini`).

## 6.4 Khac biet noi bat giua cac moi truong

- **`app.ini`**:
  - local default, `CacheService=memory`, host db/rabbit co the tro toi remote.
- **`app-local.ini` / `app-dev.ini`**:
  - thuong dung `CacheService=redis`, domain `openedudev`/`api-demo...`.
  - co bo `google_sheet_*`, `momo`, `vnpay`.
- **`app-staging.ini`**:
  - staging domains, prefix `staging_`, `IsMainnet=true` cho chain.
  - bo config `google_sheet_leaderboard_tcb_1..11` day du.
- **`app-test.ini`**:
  - test domain, prefix `test_`.
- **`app-prod.ini`**:
  - `RunMode=release`, DB SSL `require`, S3 bucket production, provider email khac.
  - nhieu gia tri de trong (co kha nang inject qua env/secret manager khi deploy).

## 6.5 Nhan xet cau hinh va van hanh

- Cau hinh env duoc phan tach ro theo file stage.
- Prefix Redis/Rabbit theo env giup tranh xung dot queue/key.
- Tuy nhien trong repo hien tai co nhieu thong tin nhay cam dang plain text trong INI (JWT, API keys, DB creds, webhook...) -> nen chuyen sang secret manager/env vars.

## 7) Mo hinh khoi tao he thong (`config.InitSystem`)

- Migrate role + permission mac dinh.
- Khoi tao user he thong + role binding.
- Khoi tao cac org mac dinh (OpenEdu, VBI, AI Gov, NDA, PDA...).
- Seed page config, forms, pricing plans (free/internal), AI models, payment methods, referral campaign settings...
- Co tinh idempotent: da ton tai thi bo qua.

## 8) Monitoring/observability

- Middleware metrics + endpoint `/metrics` (Prometheus).
- `tracing/grafana` va `tracing/prometheus` cho deployment quan sat.
- Logger setup o startup; output theo runtime path/format tu config.

## 9) Ket luan

- `openedu-core` la monolith backend nghiep vu lon, domain phong phu, da tach lop kha ro.
- API layer rat day (public + internal), RBAC va multi-tenant schema la trung tam.
- Data flow chinh: Gin -> middleware auth/tenant/permission -> service -> repository/cache/external.
- Config layer rat day du, linh hoat theo stage, nhung can uu tien hardening secret management de an toan hon.
